帮我们技术组搭一个内部博客系统，组员平时写写技术分享、踩坑记录啥的。前端 Vue 3 配 Vite，后端 Go 用 Gin 框架。数据存内存里就行，用 sync.Map 或者 mutex 包 map 都行，重启丢数据无所谓，每周手动备份一下。端口走配置，前端 5173，后端 8080。

后端 API 走这些：GET /api/articles 列表（带 page/size/tag/category/search 参数）、POST /api/articles 新建、GET/PUT/DELETE /api/articles/:id 单条操作、POST /api/articles/:id/comments 发表评论、POST /api/articles/:id/like 点赞。错误统一返回 `{error: "msg", code: 400/404/409}`，列表分页要返回 total。Markdown 渲染前后端都行——后端用 goldmark 转 HTML 存起来，前端直接展示更省事。

博客文章有标题、作者、发布时间、分类（前端/后端/运维/杂项）、标签（多选）、内容（支持 Markdown）、是否置顶这几个字段。列表页按发布时间倒序排，置顶的文章混在列表里但要明显看出来。详情页显示文章内容，底下是评论区，评论支持嵌套回复一层就够。

置顶规则我按"按置顶时间倒序排在最前，同置顶时间内的按发布时间倒序"实现，详情页加一段说明文字告诉用户当前规则。访问量是进程内计数，重启清零无所谓，每次 GET 详情接口就 +1 用 atomic.Int64 原子加，不要用 mutex 锁。

分类这块比较烦，前端和后端分类是建项目时手敲死的几个，还是运行起来能加能改？我看有的同事就希望写文章时分类可以临时加。你给个折中方案吧，但分类创建的功能一定要有，不然用户加不了新分类。分类的 CRUD 接口要做完整：GET /api/categories 列表、POST /api/categories 新建、PATCH/DELETE /api/categories/:id 改和删。删除分类前要检查该分类下是否有文章，有就拒绝删。

评论功能要做完整，能回复指定评论，支持楼中楼一层就够。评论列表按时间正序还是倒序由你定。点赞功能做不做都行，做的话要支持对文章和评论都能点，点赞数记在 article.likeCount 和 comment.likeCount 字段上，同一用户（用 cookie 临时生成 client_id 区分）只能点一次，取消点赞再点算两次。搜索功能要支持按标题、标签、内容模糊匹配，搜索结果高亮关键词。

页面风格简洁一点就行，登录注册不要做，默认所有人都是作者。整篇文章能导出成 PDF 让作者备份，最好是前端用现成库做，比如 jspdf 加 html2canvas。整站跑起来前后端都起来后能正常浏览、写作、看评论就算 OK。

附加项：博客首页除了文章流再加一个"热门标签云"模块，从所有文章的 tags 数组聚合出现频次，前 30 个标签按字体大小和颜色深浅区分热度。评论能点赞（每用户每评论一次，cookie 存 clientId 区分）。文章列表支持按作者筛选和按访问量排序。

再补：博客支持"草稿自动保存"功能，每 30 秒把编辑器内容存到 localStorage（按用户 id + 文章 id 作 key），刷新页面或意外关闭能恢复未保存内容。导出 PDF 时要带样式（标题、代码块、引用块区分），用 jspdf + html2canvas 组合。

博客首页要展示"近期活跃作者"榜单（按作者最近 30 天发文数 + 文章总访问量综合排序，前 10 名）。文章详情页要展示"相关推荐"——基于 tags 重叠度推荐同标签的 5 篇文章。评论要能"举报"——举报后管理员后台能看到举报列表并处理。

博客要支持"归档"——按月归档文章列表（侧边栏显示月份，点击展示该月所有文章）。文章详情页要展示"目录"（按标题自动生成 TOC，滚动时高亮当前章节）。

博客要支持"暗黑模式"切换，存 localStorage。文章要支持"字数统计"和"预计阅读时间"（按 200 字/分钟算）。评论要支持"@提及"——@ 后输入作者名弹提示。

博客要支持 RSS 订阅（/rss.xml 返回 RSS 2.0 格式含最新 20 篇文章）。文章详情页要展示"阅读进度条"（顶部 sticky 进度条按滚动距离算百分比）。
